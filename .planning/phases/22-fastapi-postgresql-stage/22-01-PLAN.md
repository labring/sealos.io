<interfaces>
The accepted Stage 1 public factory is `create_app() -> FastAPI`. Phase 22 extends it later with an optional explicit database URL while preserving `TaskCreate`, `TaskUpdate`, `Task`, `app`, and `create_app` as public exports. This plan creates `Base` and `TaskRecord` only for Alembic metadata; application routes remain unchanged until Plan 22-02.

The harness contract created here accepts `--session-start --state-file PATH`, `--session-stop --state-file PATH`, `--assert-clean --state-file PATH`, `--pytest-only [pytest args...]`, `--migrations-only`, `--jobs-only [--state-file PATH]`, and `--phase-gate`. A jobs-only call with a state file attaches to that active database; calls without a state file own a complete one-shot lifecycle. The caller owns the exact state path. Session start forks the cleanup-owning supervisor, waits for PostgreSQL and port-forward readiness, atomically writes a mode-0600 shell file containing quoted `RUN_ID`, `DATABASE_URL`, `TEST_DATABASE_URL`, `SUPERVISOR_PID`, and `PORT_FORWARD_PID`, prints `SESSION_READY state_file=PATH run_id=RUN_ID`, and returns to the caller. Every caller loads it with `set -a; source "$STATE_FILE"; set +a`. Session stop receives the same path and preserves it for cleanup inspection; assert-clean receives the same path, proves the recorded processes and run-labeled objects are absent, then removes the state file. A task session keeps one supervisor-owned PostgreSQL run and port-forward alive across its RED/GREEN commands.
</interfaces>
