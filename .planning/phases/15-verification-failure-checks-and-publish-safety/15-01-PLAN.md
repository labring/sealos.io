Insert a new `## If the first upload fails` section in `content/docs/guides/object-storage/index.en.mdx` directly after `## Verify Your Upload` and before `## What's Next`.

The section must be a fixed five-item first-pass checklist in this exact order (per D-08, D-09):

1. Wrong or missing bucket — bucket not created, or the wrong region/workspace selected in the console.
2. Permission denied — bucket-level permission too restrictive for your session, or the action is unavailable in the current context.
3. Wrong endpoint — internal vs external endpoint mismatch (relevant when you move to SDK or CLI access).
4. Wrong credentials — Access Key and Secret Key pair copied incorrectly (relevant when you move to SDK or CLI access).
5. Upload size or file-name issue — file too large for console upload, or characters in the file name the bucket rejects.

Checklist rules:
- One short "what to look at first" hint per bullet, one line each (D-10). Pick concrete, operational hints (e.g., "recheck the bucket list in the current workspace" rather than narrative advice).
- Checks 3 and 4 must be explicitly framed as "matters when you move to SDK or CLI access" (D-11). They must not turn into an SDK or CLI tutorial.
- Tone: short, hard, procedural. No apologies, no reassurance filler, no retry advice, no network troubleshooting tree, no decision tree (D-12).
- Do not link to a deeper troubleshooting page. The section stands on its own.
- Do not introduce code blocks, SDK examples, CLI commands, or links to non-existent Object Storage child pages.

Implements D-08, D-09, D-10, D-11, D-12 from Phase 15 CONTEXT.
