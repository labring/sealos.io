## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | 图标与短标签之间的最小间距 |
| sm | 8px | 紧凑列表、徽标、行内提示间距 |
| md | 16px | 默认段落间距、表格单元格内边距、截图与说明文字间距 |
| lg | 24px | 小节顶部留白、router 次级区块内边距 |
| xl | 32px | 页面内主要分组间距、列表组与表格之间的分隔 |
| 2xl | 48px | 大章节断点、router 主 CTA 区与后续路径区的分隔 |
| 3xl | 64px | 页面级起始留白与长页面大段落之间的分隔 |

Exceptions: `44px` minimum tap-target height for mobile sidebar rows, router CTA rows, and accordion-like disclosure triggers. 其余间距一律落在 4/8/16/24/32/48/64 体系内。

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #FFFFFF | 文档正文背景、文章主画布、表格基础底色 |
| Secondary (30%) | #F5F5F5 | 侧边栏、卡片底、信息分组容器、截图包裹层 |
| Accent (10%) | #146DFF | 仅用于主路径 CTA、当前选中路径强调、关键路由提示链接、taxonomy 锁定提示 |
| Destructive | #EF4444 | 仅用于 slug/导航替换等破坏性确认提示 |

Accent reserved for: `Start First CronJob` 主 CTA、router 页三路径里的主路径强调、故障症状入口中的当前问题高亮、需要用户跳转到下一页面的关键导航链接。不要把 accent 用到所有普通正文链接、所有表格边框或所有交互元素。

## Page Role Contract

| Surface | Route Contract | UI Role | Required Elements |
|---------|----------------|---------|-------------------|
| CronJob router page | `/docs/guides/cronjob/` via `index.en.mdx` | 入口路由页，不承载完整教程 | 1 段范围说明、`CronJob` 主命名 + `scheduled task` 辅助桥接、1 个主路径 CTA、2 个次路径入口 |
| First-success tutorial | `/docs/guides/cronjob/first-cronjob/` | 零到一 happy path | 固定示例表、编号步骤、验证清单、后续路由 |
| Combined lifecycle page | `/docs/guides/cronjob/manage-cronjob/` | 统一承载 day-2 操作 | 同页锚点：`Update a CronJob`、`Pause or Resume a CronJob`、`Delete a CronJob` |
| Troubleshooting router | `/docs/guides/cronjob/troubleshoot/` | 症状分诊页 | 3 个症状入口，每个入口 1 句症状描述 + 1 个跳转链接 |
| Symptom page 1 | `/docs/guides/cronjob/cronjob-did-not-run/` | 未触发类问题 | 现象描述、先查项、回到教程或生命周期页的恢复链接 |
| Symptom page 2 | `/docs/guides/cronjob/cronjob-ran-but-result-was-wrong/` | 结果异常类问题 | 现象描述、结果校验点、回到相应任务页的恢复链接 |
| Symptom page 3 | `/docs/guides/cronjob/cronjob-failed-after-triggering/` | 触发后失败类问题 | 失败信号、执行失败排查、回到相应任务页的恢复链接 |

Sidebar order contract:

1. `first-cronjob`
2. `manage-cronjob`
3. `troubleshoot`
4. `cronjob-did-not-run`
5. `cronjob-ran-but-result-was-wrong`
6. `cronjob-failed-after-triggering`

Router-page contract:

- `index.en.mdx` 作为 section root 保留在 `/docs/guides/cronjob/`，不要重复放入 section-local `meta.en.json` 的 `pages` 数组。
- 入口区块顺序固定为：`Start here` -> `Manage an existing CronJob` -> `Troubleshoot CronJob problems`。
- 页面应保持文档页语义，不做 dashboard、tabs、wizard、search overlay 或卡片瀑布流。

## Interaction Contract

- 保持现有 `DocsLayout` 壳层、sidebar、TOC、search 和顶部背景处理；Phase 6 只定义内容 IA，不引入新的页面级导航外壳。
- Router 页使用单列阅读流：短引导段落、一级主路径、二级路径、后续链接。不要做三栏营销卡片布局。
- 生命周期页使用同页锚点导航而不是拆成多个公开 slug；每个锚点块都必须以“什么时候用这个动作”开头，以“期望结果/验证”结尾。
- Troubleshooting router 只做分诊，不写完整修复步骤；完整修复步骤放在具体症状页。
- 症状页底部必须给出回流链接，目标只能是 `first-cronjob` 或 `manage-cronjob`，不要把用户送回模糊 overview。
- 截图在本 phase 只作为产品证据来源，不作为最终英文页面的视觉主角；如果截图内含中文 UI 文本，后续 phase 必须替换或裁切，不能直接作为英文公开页面的主图。
- 所有公开路径保持 flat slug；禁止引入 `/create/first-run/` 这类多层公开 URL。
- `CronJob` 是主展示名；仅在首次定义时用一句辅助说明引入 `scheduled task`，之后不切换主名。
